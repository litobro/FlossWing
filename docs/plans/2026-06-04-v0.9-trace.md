## Task D â€” `tools/findings.py` add `record_trace`

```python
class CallChainStep(BaseModel):
    symbol: str
    file: str
    line: int
    is_entry_point: bool
    notes: str = ""


class RecordTraceInput(BaseModel):
    finding_id: str
    reachable: Literal["reachable", "unreachable", "uncertain"]
    entry_point_symbol: str | None
    call_chain: list[CallChainStep]
    rationale: str


class RecordTraceOutput(BaseModel):
    trace_id: str


def record_trace(
    inp: RecordTraceInput,
    *,
    run_id: str,
    agent_session_id: str,
) -> RecordTraceOutput: ...
```

**Validation order:**

1. `finding_id` exists for `run_id` â†’ else `FindingNotFoundError`.
2. Finding is `status='confirmed' AND (dedupe_role IS NULL OR
   dedupe_role='primary')` â†’ else `FindingNotTraceableError`
   (defence in depth â€” stage shouldn't pass non-eligible findings).
3. No existing `traces` row for this finding â†’ else
   `TraceAlreadyExistsError`.
4. `reachable='reachable' â†’ entry_point_symbol IS NOT NULL` â†’ else
   `InconsistentTraceError`.
5. `len(call_chain) >= 1` â†’ else `EmptyCallChainError`.
6. `rationale` non-empty (after strip) â†’ else `RationaleEmptyError`.

On success, in one `session_scope()`:
- Generate `trace_id = str(ULID())`.
- Look up `entry_point_id` by `entry_point_symbol` in `entry_points`
  table (NULL if no match â€” schema permits).
- INSERT `traces` row with all fields. `call_chain_json` =
  `json.dumps([step.model_dump() for step in inp.call_chain],
  sort_keys=True)`. `created_at` ISO Z.
- UPDATE `findings.reachable = inp.reachable`.

Return `RecordTraceOutput(trace_id=trace_id)`.

**Errors** (in `flosswing/errors.py`): `RecordTraceError(FlossWingError)`
parent + 5 leaf classes. Reuse existing `FindingNotFoundError`.

## Task E â€” `stages/trace.py`

```python
@dataclass(frozen=True)
class TraceStageResult:
    outcome: Literal["completed", "skipped"]
    findings_total: int = 0
    findings_traced: int = 0
    findings_reachable: int = 0
    findings_unreachable: int = 0
    findings_uncertain: int = 0
    findings_refused: int = 0
    findings_errored: int = 0
    findings_budget_exceeded: int = 0
    input_tokens: int = 0
    output_tokens: int = 0

    @classmethod
    def skipped(cls) -> "TraceStageResult":
        return cls(outcome="skipped")


async def run(*, run_id, repo, cfg, session_factory) -> TraceStageResult:
    ...
```

**Selection:** SELECT findings WHERE
`run_id=? AND status='confirmed' AND (dedupe_role IS NULL OR dedupe_role='primary')`,
ULID order. Zero rows â†’ `TraceStageResult.skipped()`.

**Per-finding loop (sequential):**

1. `agent_session_id = str(ULID())`.
2. INSERT partial `agent_sessions` row (`stage='trace'`,
   `finding_id=<id>`, `task_id=None`). Required before
   `record_trace` runs because `traces.agent_session_id` FK is `ON
   DELETE RESTRICT`. v0.6 Validate's partial-INSERT pattern.
3. Load `prompts/system/trace.md` once; substitute `<max_depth>` =
   `cfg.trace_max_depth`.
4. Per-finding user prompt: id, file:line, function, attack_class,
   title, description, walk-backwards instruction with depth cap.
5. Build 8-tool list (re-using existing builders from
   `tools/fs.py`, `tools/search.py`, `tools/symbols.py`,
   `tools/findings.py`; `record_trace_tool(run_id, agent_session_id)`
   is the new one).
6. `await run_session(...)` with `cfg.trace_token_budget`.
7. UPDATE `agent_sessions` row with terminal fields.
8. Classify:
   - `completed` AND `traces` row exists for finding â†’ check
     `reachable` column; bucket into reachable/unreachable/uncertain;
     `findings_traced += 1`.
   - `completed` AND no `traces` row â†’ `findings_errored += 1` (agent
     didn't call `record_trace`).
   - `refused` â†’ `findings_refused`.
   - `budget_exceeded` â†’ `findings_budget_exceeded`.
   - `errored` â†’ `findings_errored`.
9. Per-finding failures swallowed; continue.
