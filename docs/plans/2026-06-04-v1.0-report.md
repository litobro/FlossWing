## Scope reminder

Pure deterministic SQLite-to-files renderer.

- **No SQLAlchemy model work.** All models Report reads are mapped
  from v0.5–v0.9.
- **No schema changes.**
- **No agent tools, no SDK calls, no new prompts.**
- **SARIF deferred to v1.1** — `--format sarif` parses but prints
  "not yet implemented".
- Single file `flosswing/stages/report.py`. Lift to package only if
  it exceeds ~600 lines.

### Functions

```python
def _load(run_id, session_factory) -> ReportV1: ...
def _render_markdown(report: ReportV1) -> str: ...
def _render_json(report: ReportV1) -> str: ...
def _write_findings_dirs(report: ReportV1, output_dir: Path) -> int: ...

@dataclass(frozen=True)
class ReportRenderResult:
    formats_written: list[str]
    findings_dirs_written: int
    output_dir: Path
    bytes_written: int
    sarif_skipped: bool

def render(*, run_id, session_factory, output_dir: Path,
           formats: list[str]) -> ReportRenderResult: ...
```

### Rendering rules

- Markdown severity sections in order: critical, high, medium, low,
  info.
- Within each severity, sort by reachability: reachable, uncertain,
  unreachable, NULL.
- UNCERTAIN findings show with explicit `[uncertain]` badge in title.
- CONFIRMED findings get a `findings/<id>/` dir with `finding.md` and
  `poc.py` (if poc_code present).
- Pending/uncertain/rejected/superseded do NOT get a dir.
- `schema_version: "1.0"` is the literal field in `report.json`.

### Injection safety

Escape user-controlled strings before rendering into markdown:
- Replace `\`\`\`` sequences inside code-fence content with `'''`.
- For text outside code fences, run through `html.escape` (or strip
  obvious HTML tags). Markdown headers (`#`) inside user content
  should be prefixed with `\` to avoid creating unexpected sections.

Document the chosen escape strategy in a module-level docstring.

### Style

- `from __future__ import annotations`
- All SQL through `session_scope()`, snapshot ORM attrs as Pydantic
  models or tuples inside the scope.
- No `# type: ignore` without inline rationale.

## Task C — Orchestrator wiring

**`flosswing/orchestrator.py`** — after `runs.status='completed'` is
committed:

```python
report_result: ReportRenderResult | None = None
if cfg.auto_render:
    output_dir = cfg.output_dir or (Path.home() / ".flosswing" / "runs" / run_id / "output")
    try:
        report_result = stages.report.render(
            run_id=run_id,
            session_factory=st_session.session_factory(),
            output_dir=output_dir,
            formats=cfg.output_formats,
        )
    except Exception as e:
        # Don't fail the run on render error; the state DB is intact.
        report_result = None
        # Print scrubbed error to summary.
```

Printed summary gains a Report section (output dir, formats,
findings dirs written, bytes). When `cfg.auto_render is False`:
print `"Report: skipped (--no-report)"`.
